### 2.2 Yazma İlkeleri
1. **Kullanıcı için yaz, ürün için değil.** "Stratejik plan oluşturmak için..." değil "Yıllık planınızı kurmak için..."
2. **Adım adım, görselli.** Her how-to: 1 ekran görüntüsü + 1 GIF
3. **Versiyonlu.** Major sürüm geçişlerinde eski sürüm doc'u korunur 18 ay
4. **Aranabilir.** Algolia DocSearch veya Mintlify native arama
5. **Çift dilli.** TR + EN her sayfa (i18n routing); diğer 21 dil otomatik çeviri + insan check (uzun vade)
6. **Geri bildirim widget'ı.** "Bu makale faydalı oldu mu?" — analytics

## 3. Product Analytics

### 3.1 İhtiyaç
- **Davranış:** Hangi özellikleri kim kullanıyor?
- **Funnel:** Onboarding tamamlama oranı?
- **Retention:** D1/D7/D30 dönüş?
- **Adoption:** Yeni özellik 30 gün içinde tenant'ların kaçında?
- **Engagement:** DAU/WAU/MAU per tenant

### 3.2 Araç Seçimi

**Karar: PostHog (self-host)**
- Açık kaynak, self-host = veri kontrolü (uyum dostu)
- Product analytics + feature flag + session replay + A/B test tek araçta
- Maliyet: Cloud $0-450/ay küçük takım; self-host yalnız altyapı
- Alternatifler reddedildi: Amplitude (kapalı, pahalı), Mixpanel (kapalı), Heap (limit)

### 3.3 Event Şeması

**Naming:** `<object>_<verb>` — `okr_checked_in`, `plan_published`, `risk_created`

**Standart properties:**
- `tenant_path`, `tenant_id`, `user_id`, `package`, `locale`, `module`, `feature`, `source` (web/mobile/api)

**Anonimleştirme:** Hiçbir PII (e-posta, isim, telefon) event payload'ında yok. user_id sözde-rastgele UUID.

### 3.4 Kritik Eventler (MVP)

**Identity & Tenant:**
- `user_signed_up`, `user_logged_in`, `user_invited`, `mfa_enrolled`
- `tenant_created`, `tenant_module_toggled`, `tenant_switched`

**Strateji:**
- `plan_created`, `plan_published`, `pillar_added`, `strategy_added`
- `okr_created`, `okr_checked_in`, `kr_updated`, `okr_cascaded`
- `hoshin_xmatrix_saved`, `catchball_round_completed`
- `efqm_assessment_completed`

**İcra:**
- `initiative_created`, `stagegate_decided` (gate, decision)

**Süreç & Risk & ESG:**
- `process_created`, `cmmi_assessed`
- `risk_created`, `kri_threshold_breached`
- `tcfd_report_generated`

**Engagement:**
- `dashboard_viewed`, `widget_added`, `report_exported`
- `ai_query_submitted`, `ai_response_rated` (helpful/not)
- `notification_clicked`, `search_performed`

### 3.5 Funnel'lar (öncelikli)
1. Sign-up → first dashboard view
2. Tenant admin invite → ilk 3 user join
3. Plan created → first OKR → first check-in
4. KPI created → ilk ölçüm
5. Initiative → ilk stage-gate
6. AI query → rated

### 3.6 Veri Yönetimi
- Event verisi 24 ay
- Aggregated metric: süresiz
- Müşteri "opt-out" hakkı (rare ama destekli — analytics yine de iç metrik için)
- Müşteri-isteğine-bağlı tenant analytics dashboard (Y2)

### 3.7 İç Dashboard (Grafana)
- Acquisition: yeni signup, conversion oranı
- Activation: ilk-değer-süresi (TTV)
- Engagement: DAU/MAU, feature reach
- Retention: cohort retention curve
- Revenue: ARR, NRR, churn (Stripe/Recurly entegrasyon)
- Product health: hata oranı per feature

### 3.8 A/B Test (Y2+)
- PostHog Experiments
- Statistical significance otomatik
- Test reach: en az 100 tenant veya 1000 kullanıcı
